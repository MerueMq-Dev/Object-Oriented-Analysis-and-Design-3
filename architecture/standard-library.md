# Стандартная библиотека

Из текущего проекта можно взять несколько идей и классов для других программ, если сделать их более общими. В первую очередь это GameStateMachine — это простой способ работать с состояниями и переходами между ними. Он подойдёт для любых программ, где есть режимы, экраны или этапы работы.

Также полезна сама роль GameController. Это главный класс, который получает события и связывает между собой разные части системы. Такой подход можно использовать почти в любой интерактивной программе.

Leaderboard тоже можно использовать повторно, если сделать из него общий класс для хранения рейтингов или лучших результатов. Такие таблицы нужны во многих задачах.

Структуру хранения поля (GameBoardStorage) можно превратить в обычную таблицу с ячейками. Такие структуры часто нужны при работе с картами, таблицами и другими данными в виде сетки.

А вот классы вроде GameBoard, GameElement слишком связаны именно с игрой «три в ряд», поэтому для общей библиотеки они не подходят.
