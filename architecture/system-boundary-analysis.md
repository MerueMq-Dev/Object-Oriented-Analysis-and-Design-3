# Анализ границ игры «Три В Ряд»

## Что будет в системе? 
- Пользовательский интерфейс (TUI) для запуска партии, выполнения ходов и просмотра результатов
- Игровое поле с разноцветными элементами
- Логика обнаружения совпадений элементов
- Система подсчёта и начисления очков
- Механизм игровых сессий
- Таблица лидеров
- Управление через клавиатуру

## Что НЕ будет в системе?
- Соревновательного режима для игры вдоём на одном устройстве
- Режима для онлайн игры сразу на нескольких устройств 
- Системы достижений и наград 
- Графического интерфейса (GUI)

## Главные подсистемы
### Подсистема обработки пользовательского ввода
В неё входит: интерфейс TUI, управление через клавиатуру, обработка команд запуска/завершения партии, обработка команд перемещения элементов, отображение игрового поля, меню и таблицы лидеров.
Подсистема отвечает за связь игрока и системы, отдельная ответственность.

### Подсистема игровой логики
В неё входит: управление состоянием поля, валидация ходов, обнаружение совпадений, расчёт очков, управление сессиями
Подсистема объединяет всю «внутреннюю механику» игры

### Подсистема сохранения результатов
В неё входит: таблица лидеров, история сессий, начисленные очки.
Подсистема имеет отдельную ответственность за хранение и предоставление данных, не смешивается с вводом или игровой логикой.

## Пользовательские метафоры
- Игра «Три в ряд» — логическая головоломка, в которой игрок взаимодействует с элементами на игровом поле с целью набора очков.
- Игровое поле (доска) — основное игровое пространство, представленное в виде сетки с размещёнными на ней элементами.
- Элемент — игровой объект, размещаемый на поле, отличающийся от других элементов по цвету или типу.
- Ход — одно действие игрока по изменению расположения элементов на игровом поле.
- Совпадение — группа одинаковых элементов, расположенных в ряд на игровом поле.
- Партия (игровая сессия) — отдельный игровой процесс от начала до завершения, в рамках которого игрок набирает очки.
- Очки — числовой показатель результата игрока в рамках партии.
- Таблица лидеров — перечень лучших игровых результатов, доступный для просмотра пользователю.


## Функциональность системы
- Запуск новой партии — пользователь может начать новую игровую партию.
- Выполнение игрового хода — пользователь может изменять расположение элементов на игровом поле.
- Определение совпадений — система поддерживает выявление совпадений элементов на поле.
- Подсчёт очков — система ведёт подсчёт очков в рамках партии.
- Обновление игрового поля — система поддерживает изменение состояния поля в процессе игры.
- Завершение партии — система поддерживает завершение игровой партии и получение результата.
- Просмотр таблицы лидеров — пользователь может просматривать результаты завершённых партий.
- Выход из игры — пользователь может завершить работу с приложением.

## Библиотеки повторного использования 
- Платформа .NET и стандартные библиотеки C#
- Библиотека Terminal.Gui для реализации TUI
- СУБД PostgreSQL для хранения результатов
- ORM-библиотека Dapper для работы с базой данных
